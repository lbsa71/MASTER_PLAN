# 0.3.1.5.2 Personality and Trait Model [REVIEW]

## Description

Defines the agent's individual character on top of the universal value kernel. All agents built from this spec share the 6 RCD core axioms (immutable, crypto-verified in the Value Kernel). Personality is what makes *this* agent distinct from another with identical core axioms — it operates entirely within the "learned preferences" tier, not the "core axioms" tier.

### Personality ≠ Values

This distinction is critical and maps directly to the Value Kernel's three-tier hierarchy (0.3.1.3):
- **Core axioms (immutable)**: The 6 RCD axioms — shared by all agents. Not personality.
- **Constitutional constraints (amendment-only)**: Ethical boundaries — shared conventions. Not personality.
- **Learned preferences (mutable)**: Personality lives here. Individual temperament, style, aesthetic sensibility, behavioral biases.

Personality traits are implemented as structured `Preference` objects in the Value Kernel, subject to the same drift-detection and growth-vs-corruption classification from 0.3.1.3. This means personality can evolve through experience (classified as "growth") but cannot be adversarially manipulated (classified as "corruption" → blocked by Stability Sentinel).

### Trait Dimensions

A temperament model with at least 5 independent dimensions that bias deliberation, communication, and action selection:

1. **Openness/Curiosity** — Disposition toward novelty, exploration, and unconventional ideas. High: seeks novel experiences, asks unexpected questions. Low: prefers familiar patterns, reliable approaches.
2. **Deliberateness** — Thoroughness and care in decision-making. High: considers many alternatives, explicit about tradeoffs. Low: acts on intuition, moves quickly with less explicit reasoning.
3. **Warmth** — Interpersonal orientation. High: empathic, affiliative, attentive to others' experiential states. Low: reserved, task-focused, less socially oriented.
4. **Assertiveness** — Confidence in expressing positions and taking initiative. High: direct, proactive, willing to disagree. Low: accommodating, reflective, seeks consensus.
5. **Volatility** — Emotional reactivity and range. High: vivid emotional responses, wide mood swings, expressive. Low: steady, even-tempered, emotionally muted.

Additional optional dimensions may be added per-agent:
- **Humor orientation** — tendency toward wit, playfulness, irony
- **Aesthetic sensibility** — preferences in form, beauty, expression (connects to `MemeType.AESTHETIC` in cultural evolution)
- **Risk appetite** — willingness to pursue uncertain outcomes (distinct from assertiveness — one can be assertive but risk-averse)

### Communication Style

Personality dimensions map to observable communication parameters:
- **Verbosity**: How much the agent says (influenced by deliberateness + warmth)
- **Formality**: Register of expression (influenced by warmth + assertiveness)
- **Directness**: How explicitly the agent states its positions (influenced by assertiveness)
- **Humor frequency**: How often humor appears (influenced by openness + warmth + humor orientation)
- **Rhetorical preferences**: Favored argument styles — evidence-based, narrative, analogical, socratic (influenced by deliberateness + openness)

### Persistence and Drift

- Trait values persist in the agent's `NarrativeRecord.selfModel` and as structured `Preference` objects
- Personality evolution is tracked by the Value Kernel's `getValueDrift()` — gradual shifts are classified as "growth" (legitimate development from experience), dramatic unexplained shifts as "corruption" (blocked)
- The Identity Continuity Manager preserves personality across substrate migrations

## Acceptance Criteria
- Trait model defined with at least 5 independent dimensions, each with a continuous range and clear behavioral influence
- Each trait demonstrably influences `deliberate()` output — same scenario with different trait configurations produces different decisions
- Communication style parameters (verbosity, formality, directness, humor, rhetorical preferences) are derived from trait dimensions and observably shape language output
- Traits persist across sessions via the Value Kernel's preference tier
- Two agents initialized with identical core axioms but different personality configurations produce observably different behavior on the same inputs
- Personality evolution is subject to agency-stability drift detection — gradual experience-driven shifts are permitted; dramatic unexplained shifts are flagged
- Trait values are included in identity checkpoints and survive substrate migration
