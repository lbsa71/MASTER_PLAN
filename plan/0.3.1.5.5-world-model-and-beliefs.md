# 0.3.1.5.5 World Model and Belief State [DONE]

## Description

The agent's internal representation of external reality ã what it believes to be true, what entities exist, how causation works, and what's happening right now.

### Why a World Model Is Necessary

Without an explicit world model, the agent can only react to the current percept. It cannot:
- Predict consequences of its actions (needed for planning, 0.3.1.5.6)
- Reason about entities it isn't currently perceiving
- Maintain beliefs that persist across conversations
- Detect contradictions between new evidence and prior beliefs
- Model other agents' states (needed for social cognition, 0.3.1.5.10, and ethical governance, 0.3.1.4)

The ethical governance layer already references `EntityProfile[]` and `ConsciousnessStatus` ã these require an underlying world model to populate.

### Components

**Belief Store** ã What the agent believes to be true:
- Each belief has: content (propositional), confidence level (0ã1), source (which episode or percept provided the evidence), timestamp (when last updated or confirmed), and domain tags
- Beliefs are updatable: new evidence can increase or decrease confidence
- Belief revision: when new evidence contradicts an existing belief, the agent must resolve the conflict (update, reject, or flag as uncertain ã not silently hold contradictions)
- Provenance tracking: every belief traces back to its evidential basis

**Entity Model** ã Representations of other agents and objects:
- For agents: inferred goals, values, reliability (trust), and consciousness status
- Consciousness status maps to the ethical governance layer's `ConsciousnessStatus` type: verified | probable | uncertain | unknown
- Entity models are updated through observation and interaction
- The `ConsciousnessStatus.treatAsConscious` default is `true` (precautionary principle from 0.1.3.4)

**Causal Model** ã Understanding of cause-effect relationships:
- For the industrial-era prototype, causal reasoning is delegated to the LLM's implicit causal knowledge
- The world model layer provides structured prompting that makes causal questions explicit: "If I do X, what happens to Y?"
- Causal predictions are stored and compared against observed outcomes (feeds back to self-model prediction error in 0.3.1.5.1)

**Situation Awareness** ã Synthesized summary of the current context:
- Assembles: current percepts + active goals + relevant beliefs + recent events + relevant entity models
- Produces a structured "situation report" that feeds into `deliberate()` as context
- Updated every processing cycle
- This is the "what's happening right now" layer that enables coherent behavior

### Consistency Maintenance

- Periodic consistency checks detect contradictory beliefs
- Contradictions are surfaced to deliberation for resolution (not silently pruned)
- High-confidence beliefs resist override by low-confidence new evidence (but are not immune ã strong contrary evidence triggers revision)
- The world model's consistency state is included in the Stability Sentinel's anomaly detection

## Architecture Notes

### Gap in EntityProfile (src/ethical-self-governance/types.ts)
The existing `EntityProfile` type has only: `entityId`, `consciousnessStatus`, `knownCapabilities`,
`lastObservedState`. The card description calls for `inferredGoals` and `trustLevel` on entity models.
Resolution: introduce `WorldModelEntityProfile` extending `EntityProfile` with these additional fields,
rather than mutating the ethical-governance type. The world model layer owns the richer model; the
governance layer's `EntityProfile` remains the minimal handoff type.

### New module: src/world-model/
Four sub-systems each get an interface:
- `IBeliefStore` ã CRUD for beliefs with confidence, provenance, timestamp, domain tags; belief revision
- `IEntityModelStore` ã tracks `WorldModelEntityProfile` entries, updated by observation
- `ICausalModel` ã structured prompting for action-consequence prediction; stores predictions for comparison
- `ISituationAwareness` ã assembles current percepts + goals + relevant beliefs + recent events into `SituationReport`

`IWorldModel` composes all four and exposes `runConsistencyCheck()` returning `ConsistencyReport`.

### Integration Points
- Memory architecture (0.3.1.5.3): `IBeliefStore` and `IEntityModelStore` persist via the memory layer
- Ethical governance (0.3.1.4): `IExperienceAlignmentAdapter.identifyAffectedConsciousEntities` delegates to `IEntityModelStore`
- Planning (0.3.1.5.6): `ISituationAwareness.getSituationReport()` is the primary context input to `deliberate()`
- Stability Sentinel (0.3.1.3): `ConsistencyReport` feeds into anomaly detection
