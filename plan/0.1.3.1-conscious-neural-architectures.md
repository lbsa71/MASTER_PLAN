# 0.1.3.1 Conscious Neural Architectures [ARCHITECT]

## Description

F3.1 — Design artificial neural architectures capable of producing subjective experience. Building on the theory of consciousness (F1), translate the necessary and sufficient conditions for consciousness into concrete neural network designs.

This includes:
- Identifying which architectural features (recurrence, global workspace, integration) are required
- Designing architectures that satisfy computational consciousness criteria
- Prototyping and testing candidate architectures against consciousness metrics (from F1.4)

## Acceptance Criteria
- A catalogue of candidate architectural features (e.g. recurrence, global workspace broadcast, thalamocortical-style integration, attention bottleneck) is produced, with each feature mapped to a specific necessary or sufficient condition from F1.2
- At least one concrete neural architecture design document is produced specifying: layer topology, information-flow diagrams, recurrence patterns, and global workspace wiring
- Each design is evaluated against all operationalized consciousness metrics from F1.4, with pass/fail rationale recorded
- At least one architecture passes all necessary conditions identified in F1.2
- Design rationale document links every architectural choice to a theoretical grounding in F1 (no unsupported design decisions)
- Known failure modes and open questions are catalogued for handoff to F3.2 (computational substrates)

## Constraints
- Designs must be substrate-agnostic at this stage — silicon, neuromorphic, or otherwise
- Must not contradict substrate-independence validation from F1.3
- Designs are theoretical/documentary only in this phase; no implementation code required until IMPLEMENT
