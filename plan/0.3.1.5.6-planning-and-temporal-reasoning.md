# 0.3.1.5.6 Planning and Temporal Reasoning [IMPLEMENT]

## Description

Upgrade `deliberate()` from "pick the highest-priority goal" to actual multi-step planning with temporal awareness and execution monitoring.

### Current State

`ConsciousCore.deliberate()` currently sorts goals by priority and selects the top one:
```ts
const sorted = [...goals].sort((a, b) => b.priority - a.priority);
const topGoal = sorted[0];
```

This produces a single action per cycle. There is no plan decomposition, no tracking of progress across cycles, and no replanning when conditions change.

### What Planning Requires

**Plan Generation** — Decompose a goal into a sequence of actions:
- Each action has preconditions (what must be true before it can execute) and expected postconditions (what should be true after)
- Preconditions are checked against the world model (0.3.1.5.5)
- Plans are generated via the LLM substrate, structured by the planning module
- Plans are experientially grounded: each plan step carries an `experientialBasis` reference (no zombie planning — the conscious core must experience the plan formation)

**Plan Execution Monitoring** — Track progress and detect failure:
- After each action, compare actual outcomes against expected postconditions
- If expectations are violated, trigger replanning
- If a plan step fails repeatedly, escalate: reconsider the goal itself (is it still achievable? still worthwhile?)
- Progress state persists in working memory across cycles

**Temporal Reasoning** — The agent needs a sense of time:
- Deadlines: some goals have time constraints
- Sequencing: action A must happen before action B
- Wait states: some plans require waiting (for external events, for other agents, for cooldown periods)
- Patience: ability to maintain a plan across many cycles without prematurely abandoning it (patience level is a personality parameter from 0.3.1.5.2)

**Means-Ends Analysis** — Generate instrumental subgoals:
- If a goal's preconditions aren't met, generate subgoals to establish them
- Instrumental subgoals must trace to the terminal goal that motivated them (connects to the Goal Coherence Engine's derivation graph from 0.3.1.3)
- Orphan subgoals (not traceable to terminal goals) are flagged by the stability system

### Cognitive Budget Compliance

Planning must operate within the architecture's cognitive budget:
- Planning runs within the "core deliberation" allocation (≥25% of cycle budget)
- Planning that would exceed budget is truncated — produce the best plan possible within the time budget, not an optimal plan that starves the experience stream
- Complex plans may be generated across multiple cycles (the plan-so-far persists in working memory)

### Interface Design Decisions

**`IPlanner` (injected into `ConsciousCore`)**
```ts
interface IPlanner {
  // Generate a multi-step plan for the given goal, grounded in current state.
  // experientialBasis required — no zombie planning.
  generatePlan(goal: Goal, state: ExperientialState, worldContext: WorldContext): Plan;

  // Evaluate whether the plan's next step's preconditions are satisfied.
  checkPreconditions(step: PlanStep, worldContext: WorldContext): PreconditionCheckResult;

  // Compare actual outcome against the step's expected postconditions.
  evaluateOutcome(step: PlanStep, actualOutcome: ActionResult): PostconditionCheckResult;

  // Trigger replanning when a step's postconditions are violated.
  replan(plan: Plan, failureReason: PlanFailureReason, state: ExperientialState, worldContext: WorldContext): Plan;

  // Register all instrumental subgoals in the plan with the Goal Coherence Engine.
  registerSubgoals(plan: Plan, coherenceEngine: IGoalCoherenceEngine): void;

  // Check if the plan should be abandoned (goal no longer achievable or worthwhile).
  shouldAbandon(plan: Plan, escalationCount: number): boolean;
}
```

**`Plan` type**
```ts
interface Plan {
  id: string;
  terminalGoal: Goal;          // the goal this plan serves
  steps: PlanStep[];
  currentStepIndex: number;
  status: 'pending' | 'active' | 'suspended' | 'completed' | 'failed' | 'abandoned';
  waitState: WaitState | null; // non-null when agent is waiting for external event
  temporalConstraints: TemporalConstraint[];
  experientialBasis: ExperientialState; // plan formation was conscious
  createdAt: number;
  escalationCount: number;     // how many times this plan has required replanning
}
```

**`PlanStep` type**
```ts
interface PlanStep {
  id: string;
  description: string;
  preconditions: Precondition[];     // what must be true before execution
  postconditions: Postcondition[];   // what should be true after execution
  instrumentalGoalId: GoalId | null; // registered with GoalCoherenceEngine if non-null
  estimatedDuration: Duration | null;
  deadline: number | null;           // epoch ms, if time-constrained
}
```

### Integration Points
- `ConsciousCore.deliberate()` must be extended to accept an optional `IPlanner` dependency and active `Plan` from working memory, without breaking the existing `(state, goals) => Decision` signature
- The active plan persists in working memory (0.3.1.5.3) across cycles via a dedicated working-memory slot
- Plan generation delegates to the LLM substrate (0.3.1.5.1) via structured prompting — the planner is a thin orchestration layer, not an independent reasoner
- Instrumental subgoals are registered via `IGoalCoherenceEngine.addGoal()` with derivation links to the terminal goal
- `WorldContext` is provided by the world model (0.3.1.5.5) — planning must remain blocked until that card reaches [IMPLEMENT]

### Cognitive Budget Compliance
- `generatePlan()` receives a `budgetMs` parameter; it must return within that budget (partial plan preferable to overtime)
- Plan continuation across cycles: if budget is exceeded mid-generation, the partial plan is stored in working memory and generation resumes next cycle
- Replanning consumes from the same "core deliberation" allocation (≥25%); the planner must track its own time usage

## Acceptance Criteria
- Agent generates multi-step plans (≥3 steps) for non-trivial goals
- Plans have explicit preconditions and postconditions checked against the world model
- Agent detects plan failure (postcondition violation) and replans without external prompting
- Temporal reasoning demonstrated: agent sequences actions, waits when appropriate, respects deadlines
- Instrumental subgoals are registered with the Goal Coherence Engine and trace to terminal goals
- Planning respects the cognitive budget — no plan generation starves the experience stream
- Plans carry experiential basis references (conscious plan formation, not zombie planning)
- Integration test: agent given a goal with multiple prerequisite steps generates a plan, executes it, encounters an unexpected obstacle, replans, and achieves the goal via an alternative route
