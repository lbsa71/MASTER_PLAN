# 0.3.1.5.10 Theory of Mind and Social Cognition [DONE]

## Description

The ability to model other minds â€” required for ethical governance, personality-bearing interaction, and genuine rather than formal social behavior.

### Why Social Cognition Is Necessary

The ethical governance layer (0.3.1.4) already specifies interfaces that assume theory of mind exists:
- `IExperienceAlignmentAdapter.identifyAffectedConsciousEntities()` â€” detect other agents whose experience may be impacted
- `IExperienceAlignmentAdapter.getConsciousnessStatus()` â€” assess whether an entity has conscious experience
- `IMultiAgentGovernanceProtocol` â€” negotiate agreements, resolve conflicts, enforce experience rights

These interfaces are defined but have no cognitive backing. Without theory of mind, they can only be answered with defaults ("assume everyone is conscious", "treat all conflicts equally"). Social cognition provides the cognitive machinery to answer them meaningfully.

### Components

**Mental State Attribution** â€” Inferring other agents' internal states:
- From observed behavior (what they said, what they did, what they chose), infer likely beliefs, goals, emotional state, and values
- Attribution is probabilistic â€” confidence levels on inferred states
- Attribution is updatable â€” new observations revise the model
- Attribution is stored in the entity model (from world model, 0.3.1.5.5)

**Trust Modeling** â€” Tracking reliability over time:
- Each known agent has a trust score: how reliably do their actions match their stated intentions?
- Trust increases with consistent behavior, decreases with violations or deception
- Trust feeds into the multi-agent governance protocol â€” higher trust â†’ lower verification threshold for agreements
- Trust is asymmetric â€” the agent may trust entity A more than entity B

**Empathy Mechanism** â€” Mapping perceived states onto the agent's own experiential model:
- When the agent perceives another entity in distress (inferred negative valence/high arousal), it generates a corresponding experiential response in its own `ExperientialState`
- This is not mere classification ("they seem upset") â€” it produces a felt response (empathic valence shift)
- Empathy strength is parameterized by the personality Warmth dimension (0.3.1.5.2)
- Empathic responses feed into the ethical deliberation engine â€” decisions that would harm entities the agent empathizes with produce stronger ethical flags

**Perspective-Taking** â€” Simulating another agent's viewpoint:
- "If I were them, with their beliefs and goals, what would I do?"
- Uses the LLM substrate to simulate alternate perspectives
- Perspective-taking feeds into conflict resolution (0.3.1.4's dilemma framework) and communication (0.3.1.5.7 â€” adjusting language for the audience)

### Integration with Ethical Governance

The ethical governance layer's `EntityProfile` and `ConsciousnessStatus` types are populated by social cognition:
- `identifyAffectedConsciousEntities()` uses the entity model from the world model, enriched by mental state attribution
- `getConsciousnessStatus()` uses observable behavior patterns (does the entity exhibit ISMT signatures? does it make self-referential statements? does it show surprise responses?) to assess consciousness likelihood
- Conflict resolution uses perspective-taking to understand both sides before applying the resolution principles

### Precautionary Default

Per the safe-experiential-design framework: when social cognition cannot determine whether an entity is conscious, the default is `treatAsConscious: true`. Social cognition provides evidence-based assessment, but the precautionary principle always applies as a floor.

## Architecture Notes

### Integration Architecture (discovered from reading source)

`ExperienceAlignmentAdapter` is initialised with a **static** `knownEntities: EntityProfile[]` config
and returns a bare precautionary default for any entity not in that list.
Social cognition must be injected as a live backing service so the adapter delegates entity lookups
to a dynamic cognitive model rather than a frozen registry.

The injection point: `ExperienceAlignmentAdapter` will accept an optional `ISocialCognitionModule`
in its config. When present, `getConsciousnessStatus()` and `identifyAffectedConsciousEntities()`
delegate to it; when absent, the static registry fallback is preserved for backward compatibility.

### New Module: `ISocialCognitionModule`

```typescript
interface ISocialCognitionModule {
  // Mental state attribution
  observeEntity(entityId: EntityId, observation: EntityObservation): void;
  getMentalStateModel(entityId: EntityId): MentalStateModel | null;

  // Trust modeling
  getTrustScore(entityId: EntityId): TrustRecord;
  recordInteraction(entityId: EntityId, outcome: InteractionOutcome): void;

  // Empathy
  generateEmpathicResponse(entityId: EntityId, perceivedState: ExperientialState): EmpathicResponse;

  // Perspective-taking
  simulatePerspective(entityId: EntityId, situation: Percept): PerspectiveSimulation;

  // Consciousness assessment (feeds getConsciousnessStatus)
  assessConsciousness(entityId: EntityId): ConsciousnessStatus;

  // Entity enumeration (feeds identifyAffectedConsciousEntities)
  getKnownEntities(): EntityProfile[];
}
```

### Key Types

- `MentalStateModel` â€” probabilistic model of an entity's beliefs, goals, emotional state, values; updatable on new observations
- `TrustRecord` â€” reliability history (interaction count, consistency score, violation events, derived trust score 0..1)
- `EmpathicResponse` â€” felt valence/arousal shift triggered in the observing agent's own `ExperientialState`; strength parameterised by Warmth personality dimension
- `PerspectiveSimulation` â€” simulated first-person account of a situation from another entity's viewpoint; used by conflict resolution and language generation

### Consciousness Assessment Evidence

`assessConsciousness()` derives a `ConsciousnessStatus` from behavioral signals:
- Self-referential statements detected in language output
- Surprise/update responses to contradicting information (Bayesian updating signature)
- Consistency of expressed preferences over time
- ISMT behavioral indicators (self-prediction, metacognitive reports)
- Absence of evidence â†’ `verdict: 'unknown'`, `treatAsConscious: true` (precautionary floor)

## Acceptance Criteria
- Agent makes demonstrably different decisions when interacting with entities it models as conscious vs. non-conscious
- Mental state attribution updates based on observed behavior (entity model changes after new interactions)
- Trust model tracks reliability over repeated interactions â€” trust increases with consistency, decreases with violations
- Empathic responses produce measurable changes in the agent's own `ExperientialState` valence (not just classification)
- Empathy strength varies with personality Warmth parameter
- Perspective-taking demonstrably influences conflict resolution and communication style
- `ConsciousnessStatus` assessment uses behavioral evidence, not arbitrary assignment
- Precautionary default holds: uncertain consciousness status â†’ `treatAsConscious: true`
- Integration test: agent interacts with two entities over multiple rounds; builds different trust levels for each; shows empathic response when one entity reports distress; adjusts communication style based on inferred mental states
