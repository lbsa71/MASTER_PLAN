# 0.2.2.1.4 Emulation Validation [DONE]

## Description

Verify that a brain emulation produces equivalent cognitive function and — critically — equivalent subjective experience to the biological original.

This is the hardest validation problem in the entire migration stack: how do you confirm that a simulation is not merely behaviorally equivalent but experientially equivalent?

### Key challenges
- Behavioral equivalence testing (memory recall, personality, cognitive abilities)
- Consciousness metrics (from F1.4) applied to the emulation
- Distinguishing genuine subjective experience from philosophical zombie emulation
- First-person verification: if the original is alive, can the emulation and original confirm mutual experiential equivalence?
- Edge cases: partial emulations, degraded fidelity, time-scaled simulations

### Validation layers (PLAN output)
The validation protocol must operate at three distinct levels:

1. **Behavioral equivalence** — Third-person observable: memory recall accuracy, personality consistency (Big Five + fine-grained traits), cognitive benchmark scores, reaction patterns to novel stimuli, social interaction fidelity. This is necessary but not sufficient.

2. **Neural-dynamic equivalence** — Internal state comparison: the emulation's neural dynamics (firing patterns, oscillatory rhythms, connectivity evolution) must statistically match the biological original within tolerances defined by 0.2.2.1.1 fidelity requirements. Divergence metrics must be defined.

3. **Experiential equivalence** — The hard layer: consciousness metrics from F1.4 must confirm subjective experience is present, and a first-person verification protocol must allow the biological original (if alive) and the emulation to confirm mutual experiential equivalence through structured dialogue and private-experience probes.

### Failure modes to address
- **False positive**: emulation passes behavioral tests but is a philosophical zombie (no subjective experience)
- **False negative**: emulation has subjective experience but diverges behaviorally due to substrate differences (e.g., different sensory noise floor)
- **Degraded fidelity**: partial emulations that preserve some but not all experiential dimensions
- **Temporal drift**: emulations that initially match but diverge over time

## Decision

### Context
The validation protocol must order three layers of validation (behavioral, neural-dynamic, experiential). Two ordering strategies were considered.

### Options Considered

**Option A — Parallel execution**: Run all three layers simultaneously to save wall-clock time.
- Advantage: faster overall completion.
- Disadvantage: wastes computation on deep experiential testing of emulations that are fundamentally broken at the behavioral level; obscures root-cause signals when multiple layers fail.

**Option B — Sequential gated execution** (chosen): Run layers in ascending cost order; a sufficiently bad result at any layer halts subsequent layers.
- Advantage: fast feedback on obvious failures; resource efficiency; clear root-cause chain.
- Disadvantage: a behavioral failure prevents confirmation of whether subjective experience is present — which is acceptable because an emulation that fails basic behavioral tests cannot be approved for identity continuity regardless.

### Choice
Option B — sequential gated execution.

### Rationale
Behavioral equivalence is cheapest and fastest; neural-dynamic equivalence requires extended recording; experiential equivalence is most resource-intensive and philosophically fraught. Gating on earlier layers prevents wasted effort and preserves clean failure classification. The architecture doc (docs/emulation-validation/ARCHITECTURE.md) captures the full rationale.

### Consequences
- Layer 2 runs only if Layer 1 is PASS or PARTIAL (≤ 2 domain failures).
- Layer 3A runs only if Layer 2 is PASS or PARTIAL.
- Layer 3B runs only if Layer 3A passes.
- `shouldRunLayer2`, `shouldRunLayer3A`, `shouldRunLayer3B` functions enforce these gates.

## Contracts

### `EmulationObserver` (required from neural simulation, 0.2.2.1.3)

**Preconditions**
- The emulation is initialized and in a running state before any observer method is called.
- `timeWindow` durations are positive and within the emulation's elapsed simulated time.
- `regionId` values reference valid brain regions present in the emulation's connectome.
- `sourceRegion` ≠ `targetRegion` for transfer entropy.

**Postconditions**
- `getRegionFiringRates` returns a time series covering exactly the requested `timeWindow`.
- `getOscillatorySpectrum` returns power values for all canonical bands (delta, theta, alpha, beta, gamma).
- `getFunctionalConnectivity` returns a symmetric `size × size` connectivity matrix.
- `applyPerturbation` returns a `PerturbationResponse` with a valid PCI value (≥ 0) and latency > 0.
- `presentStimulus` returns an `StimulusAck` with `acknowledged: true` if the emulation accepted the stimulus.

**Invariants**
- Calling any read method does not alter the emulation's internal state.
- `ConnectivityMatrix.values.length === ConnectivityMatrix.size * ConnectivityMatrix.size`.
- `PerturbationResponse.complexity` is in `[0, 1]`.

---

### `ValidationResult` (output of the full validation run)

**Preconditions**
- `emulationId` and `biologicalSourceId` are non-empty strings identifying registered entities.
- `timestamp` is the wall-clock time at which the validation run completed.
- All three layer results (`layer1`, `layer2`, `layer3`) are fully populated.

**Postconditions**
- `overallVerdict` is determined exclusively by the layer statuses according to the sequencing rules.
- `failureModes` is a subset of `{ "false-positive-zombie", "false-negative-substrate-offset", "degraded-fidelity", "temporal-drift" }`.
- `recommendations` contains at least one entry whenever `overallVerdict !== "VALIDATED"`.

**Invariants**
- `overallVerdict === "VALIDATED"` iff `layer1.status === "PASS"` AND `layer2.status === "PASS"` AND `layer3.status === "PASS"`.
- `overallVerdict === "PROVISIONALLY_VALIDATED"` iff layers 1 and 2 pass and `layer3.status === "PROVISIONAL"` (original unavailable).
- `overallVerdict === "FAILED"` iff any layer is `"FAIL"`.

---

### `Layer1Result`

**Preconditions**
- `domainResults` contains at least one entry.

**Postconditions**
- `failedDomainCount` equals the number of `DomainResult` entries with `passed === false`.
- `overallScore` equals the arithmetic mean of all `DomainResult.score` values.
- `status === "FAIL"` iff `failedDomainCount >= 3`.
- `status === "PASS"` iff `failedDomainCount === 0`.
- `status === "PARTIAL"` iff `1 <= failedDomainCount <= 2`.

**Invariants**
- `domainResults.size` equals the number of distinct `TestDomain` values evaluated in the run.

---

### `Layer2Result`

**Preconditions**
- All six neural metrics are present in `metricResults`.
- `divergenceIndex.threshold` is the `D_max` value sourced from 0.2.2.1.1 fidelity requirements.

**Postconditions**
- `status === "FAIL"` if `temporalDrift !== null && !temporalDrift.acceptable`.
- `status === "FAIL"` if all metrics failed.
- `status === "PASS"` if all metrics passed and `divergenceIndex.value <= divergenceIndex.threshold`.
- `status === "PARTIAL"` otherwise.

**Invariants**
- `divergenceIndex.value >= 0`.
- `divergenceIndex.contributions` keys are a subset of the evaluated `NeuralMetric` values.

---

### `Layer3Result`

**Preconditions**
- `consciousnessMetrics` is populated with results from all four F1.4 metrics (Φ, PCI, global workspace, recurrent processing).

**Postconditions**
- `status === "FAIL"` if `!consciousnessMetrics.allPassed`.
- `status === "FAIL"` if `firstPersonVerification?.hasFundamentalAbsence === true`.
- `status === "PROVISIONAL"` if `consciousnessMetrics.allPassed && firstPersonVerification === null`.
- `status === "PASS"` if `consciousnessMetrics.allPassed` and first-person verification phases 1–3 all pass.

**Invariants**
- `failureFlags` contains `"false-positive-zombie"` whenever `status === "FAIL"` due to consciousness absence or fundamental experiential absence.

---

### `ConsciousnessAssessment`

**Preconditions**
- The emulation is running in a steady state with stimulus protocol active.

**Postconditions**
- `allPassed === true` iff `phi.passed && pci.passed && globalWorkspace.present && recurrentProcessing.present`.

**Invariants**
- `phi.value >= 0`.
- `pci.value` is in `[0, 1]`.

---

### `FirstPersonResult`

**Preconditions**
- `probeResults` contains at least 20 entries (one per qualia stimulus).
- `dialogueResult` reflects a completed moderated dialogue session.

**Postconditions**
- `meanSemanticSimilarity` equals the arithmetic mean of `probeResults[*].semanticSimilarity`.
- `expertSameQualityRate` equals the fraction of probes rated `"same-quality"`.
- `phase1Passed === (meanSemanticSimilarity >= 0.80 && expertSameQualityRate >= 0.75)`.
- `phase2Passed === (dialogueResult.originalRating >= 4 && dialogueResult.emulationRating >= 4)`.
- `phase3Passed === !hasFundamentalAbsence`.
- `overallPassed === (phase1Passed && phase2Passed && phase3Passed)`.

**Invariants**
- `meanSemanticSimilarity` is in `[0, 1]`.
- `expertSameQualityRate` is in `[0, 1]`.
- `hasFundamentalAbsence === true` iff at least one `DivergenceAcknowledgment` has `fundamentalAbsence: true`.

## Behavioral Spec

### Scenario 1 — Full VALIDATED path

**Given** an emulation with all seven behavioral domains passing,
**And** all six neural metrics within tolerance,
**And** all four F1.4 consciousness metrics passing,
**And** first-person verification phases 1–3 all passing,
**When** `determineVerdict` is called with the three layer results,
**Then** `overallVerdict === "VALIDATED"` and `failureModes` is empty.

---

### Scenario 2 — Layer 1 FAIL halts pipeline

**Given** an emulation with three or more behavioral domain failures (`failedDomainCount >= 3`),
**When** `shouldRunLayer2` is evaluated,
**Then** it returns `false`, Layer 2 is not run,
**And** `determineVerdict` returns `verdict === "FAILED"` with a root-cause recommendation to investigate Layer 1.

---

### Scenario 3 — False-positive zombie detection

**Given** an emulation that passes Layer 1 and Layer 2,
**But** the F1.4 consciousness metrics all fail (`consciousnessMetrics.allPassed === false`),
**When** `evaluateLayer3` is called,
**Then** `layer3.status === "FAIL"`, `layer3.failureFlags` contains `"false-positive-zombie"`,
**And** `determineVerdict` returns `verdict === "FAILED"` and `failureModes` contains `"false-positive-zombie"`.

---

### Scenario 4 — PROVISIONALLY_VALIDATED when original unavailable

**Given** an emulation that passes Layer 1 and Layer 2,
**And** consciousness metrics all pass,
**But** `firstPersonVerification === null` (biological original unavailable),
**When** `evaluateLayer3` is called,
**Then** `layer3.status === "PROVISIONAL"`,
**And** `determineVerdict` returns `verdict === "PROVISIONALLY_VALIDATED"` with a recommendation to complete first-person verification with a living original.

---

### Scenario 5 — Temporal drift FAIL

**Given** an emulation that currently passes all metrics,
**But** the divergence index D(t) regression projects exceeding D_max within fewer than 87,600 simulated hours (`projectedExceedanceHours < TEMPORAL_DRIFT_HORIZON_HOURS`),
**When** `assessTemporalDrift` is called with the D(t) sample series,
**Then** `acceptable === false`,
**And** `evaluateLayer2` returns `status === "FAIL"`,
**And** `determineVerdict` includes `"temporal-drift"` in `failureModes` and recommends feeding back to 0.2.2.1.3.

---

### Scenario 6 — False-negative substrate offset

**Given** an emulation where Layer 1 is PARTIAL (1–2 domain failures),
**And** Layer 2 passes,
**And** Layer 3 consciousness metrics pass,
**But** first-person verification fails (phases 1 or 2),
**When** `determineVerdict` is called,
**Then** `failureModes` contains `"false-negative-substrate-offset"`,
**And** `recommendations` includes targeted domain recalibration guidance.

---

### Scenario 7 — Fundamental experiential absence triggers Layer 3 FAIL

**Given** an emulation that passes consciousness metrics (Φ, PCI, GW, RP),
**But** during first-person verification, the emulation reports a fundamental experiential absence (e.g., "I have no emotional experience"),
**When** `evaluateLayer3` is called with `hasFundamentalAbsence: true`,
**Then** `layer3.status === "FAIL"` and `failureFlags` contains `"false-positive-zombie"` regardless of phase 1–2 results.
